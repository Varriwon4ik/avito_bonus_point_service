# LLM Usage Report — Week 3

AI / LLM tools were used during Assignment 3. This report discloses how.

## Tools used

- **Claude Code (Anthropic, Claude Opus)** — used as an engineering and
  documentation assistant inside the team's IDE/terminal.

## How they were used

- **Repository audit:** scanning issues, PRs, and files to reconcile the current
  Product Backlog state against the Assignment 3 requirements.
- **Backlog & process artifacts:** drafting `docs/user-stories.md`,
  `docs/definition-of-done.md`, `docs/roadmap.md`, `CHANGELOG.md`, the GitHub
  issue templates, and the extended PR template.
- **Report drafting:** drafting the Week 3 report structure, the customer-review
  summary and transcript clean-up, the reflection, and this retrospective.
- **GitHub workflow setup:** generating the `gh` CLI commands used to create the
  Sprint milestone, labels, MVP-version tracking, supporting PBIs, the project
  board, and the SemVer release.

## Human oversight

- All scope decisions (MVP v1 contents, `Won't Have`/removed classifications,
  Sprint Goal, ownership, reviewers) were made by the team, not the model.
- The customer review, its feedback, and approvals are the team's own record.
- Team members reviewed and edited generated content before committing it. The
  submission reflects original team analysis and effort; the LLM accelerated
  drafting and mechanical workflow steps.
